OpenStack Juno : Boot Instances
2015/01/06
 
Create and Start Virtual Machine Instance.
[1] Specify the flavor (memory or disk) and create an instance and boot it.
# default flavor list

[root@dlp ~(keystone)]#
nova flavor-list

| 1  | m1.tiny   | 512       | 1    | 0         |      | 1     | 1.0         | True      |
| 2  | m1.small  | 2048      | 20   | 0         |      | 1     | 1.0         | True      |
| 3  | m1.medium | 4096      | 40   | 0         |      | 2     | 1.0         | True      |

[root@dlp ~(keystone)]#

[3] If you'd like to stop an instance, it's also possible to control with nova command like follows.
[root@dlp ~(keystone)]#
nova list 

+-----------+-------------+--------+------------+-------------+--------------------+
| ID        | Name        | Status | Task State | Power State | Networks           |
+-----------+-------------+--------+------------+-------------+--------------------+
| ee692a0f- | Fedora_21   | ACTIVE | -          | Running     | network01=10.1.0.2 |
+-----------+-------------+--------+------------+-------------+--------------------+
# stop instance

[root@dlp ~(keystone)]#
nova stop Fedora_21

[root@dlp ~(keystone)]#
nova list 

+-----------+-----------+---------+------------+-------------+--------------------+
| ID        | Name      | Status  | Task State | Power State | Networks           |
+-----------+-----------+---------+------------+-------------+--------------------+
| 29d99611- | Fedora_21 | SHUTOFF | -          | Shutdown    | network01=10.1.0.2 |
+-----------+-----------+---------+------------+-------------+--------------------+
# start instance again

[root@dlp ~(keystone)]#
nova start Fedora_21

[root@dlp ~(keystone)]#
nova list
