MariaDB Replication
2015/11/30
 
Configure MariaDB Replication settings. This configuration is general Master-Slave settings.
[1] Change settings and create a user for replication on MariaDB Matser Host.
www:~ #
vi /etc/my.cnf
# line 28: uncomment

log-bin=mysql-bin
# line 37: change to an ID you like

server-id=
101
www:~ #
systemctl restart mysql

www:~ #
mysql -u root -p

Enter password:
Welcome to the MariaDB monitor.  Commands end with ; or \g.
Your MariaDB connection id is 3
Server version: 10.0.20-MariaDB-log openSUSE package

Copyright (c) 2000, 2015, Oracle, MariaDB Corporation Ab and others.

Type 'help;' or '\h' for help. Type '\c' to clear the current input statement.
# create user (set any password for 'password' section)

MariaDB [(none)]>
grant replication slave on *.* to replica@'%' identified by 'password';

Query OK, 0 rows affected (0.00 sec)
MariaDB [(none)]>
flush privileges;

Query OK, 0 rows affected (0.00 sec)
MariaDB [(none)]>
exit

Bye
[2]
Install and start MariaDB Server on Slave Host, refer to here.
[3] Change settings on Slave Host.
node01:~ #
vi /etc/my.cnf
# line 28: uncomment

log-bin=mysql-bin
# line 37: change to an ID you like (different one from Master Host)

server-id=
102
# line 38: add

read_only=1
# define own hostname

report-host=node01.srv.world
node01:~ #
systemctl restart mysql

[4] Get Dump-Data on MariaDB Master Host.
www:~ #
mysql -u root -p
